# 亞里庫島 / 虛擬寵物記帳 — 專案交接筆記

> 給下一個 session 的快速上手文件。狀態都在檔案裡，讀完這份 + 掃一下 `index.html` 即可接手。

## 一句話
GameFi 浮島「虛擬寵物記帳」App：記帳理財 × 電子雞養成 × 股市小遊戲。單檔 vanilla JS（**不是 React**），像素風，可部署到 Render。

## 檔案結構
- `index.html` — 主程式，HTML/CSS/JS 全在裡面（~6000 行）。所有 UI 與邏輯。
- `assets.js` — base64 素材常數（圖片、sprite），在 index.html 的 script 之前載入，全域可用。
  - 重要常數：`TIGER_SPRITE_SRC`/`TIGER2/3`（老虎動畫）、`PET_IMG`（各生肖靜態圖）、`EXPENSE_ICON_SRC`（分類圖標 4×3 sprite）、`PIGGY_MASK_IMG`/`PIGGY_SHELL_IMG`（液體豬遮罩/外框）、`CHAT_ICON`、`WAR_ICON`、`MONEY_IMG`、`NAV_ICON`、`ISLAND_IMG`、`FOODBAG_IMG` 等。
- `server.js` — Node 零框架…**更正：用 express**。靜態伺服 + `/api/chat`（Gemini 代理）+ 掛載 `routes/`。綁 `process.env.PORT`（Render 相容）。
- `lib/` — `auth.js`（JWT/Google 驗證）、`db.js`、`users.js`、`zodiac.js`。
- `routes/photos.js` — `/api/photos/presign`（S3 presigned 上傳骨架，未設 S3 時回 deferred）。
- `.env`（**gitignore，勿 commit**）：`GEMINI_API_KEY`、`GEMINI_MODEL`、`GOOGLE_CLIENT_ID`、`JWT_SECRET`、`JWT_EXPIRES`、`PORT`。
- `ui source/` — 原始美術圖（處理後多半 base64 進 assets.js）。

## 開發/驗證慣例（重要）
- 改完 index.html 一定跑：`node -e "require('vm').compileFunction(require('fs').readFileSync('index.html','utf8').match(/<script>([\\s\\S]*)<\\/script>/)[1],[],{})"` 確認語法。
- 改 assets.js 用 vm 在 sandbox context eval 驗證常數存在。
- 圖片處理（去背/切圖/sprite）用 Python PIL + scipy，處理完 base64 進 assets.js，暫存檔記得刪（刪不掉時用 cowork 的 allow_cowork_file_delete）。
- **VM 路徑**：專案在 sandbox 是 `/sessions/.../mnt/虛擬寵物記帳/`。

## 主要系統與關鍵函式
- **狀態** `S`（localStorage key `gamefi_island_ledger_v1`）：`fresh()` 初始、`load()`、`save()`。雲端同步鍵 `GAME_STATE_KEYS`。
- **分頁**：`switchTab(t)`、`TABS={home,ledger,war,shop,stats,settings}`、`updateTabChrome`。底部 sheet `#sheetBody`。
- **寵物**：`petAnim`（首頁老虎，tiger.png 走路/跳/睡幀）；只有**老虎有完整動畫**，其他生肖靜態圖。`setupPetDisplay()`、`petLoop()`、`isTigerSprite()`。場景（餐廳/寢室）用 `tiger4.png` + `scenePet`。
- **PetRig**（跨物種抽象層）：`PetRig.setZodiac/setSkin/equip/setMood`。`backend='sprite'|'spine'`。⚠️ 登入會設 `backend='spine'` → `setupPetDisplay` 對**非老虎**走 `ZodiacSpine`（讀 base_zodiac_pet.json，平面圖浮動）；**老虎一律走真 sprite**（別再讓 spine 蓋掉老虎動畫）。
- **記帳**：`renderLedger()` 有子分頁 `ledgerSub='quick'|'dream'`。
  - 快速記帳 `renderQuickLedger()`：支出/收入、金額（懸浮數字鍵盤 `openLedgerNumpad`，有「＋再記一筆」`onAddMore`）、分類 chip（像素圖標 `catIcon(name,fallback)` + `CAT_SPRITE` map）、時光寶箱、歷史紀錄按鈕。
  - 圓夢浮島 `renderDreamLedger()`：多目標 `S.savingGoals`，垂直卡片列表 `.dream-scroll-list`，`＋新增圓夢目標` 在底部。安全消費算法 `recalcDreamBudget()`/`computeGoalStats()`。卡片左側=**液體流動存錢豬** `dynamicPiggy(progress)`（遮罩 PIGGY_MASK + 液體水位 + 波浪 + 氣泡；100% 發光浮動）。
- **時光寶箱／相片（Cursor 最新改）**：App 內即時相機 `startTimeBoxCamera`/`displayTimeBoxPhoto`，相片存成 asset 物件（`photoBlobRegistry` 存 blob），`finalizePhotoAsset`/`releasePhotoAsset`，顯示用 `photoDisplaySrc(photo)`。相片牆 `openPhotoWall()`。上傳走 `/api/photos/presign`（未設 S3 → 本機暫存）。
- **聖戰 2.0**（股市小遊戲）：2 兵種（多頭步兵/空頭刺客）、10 虛擬股 `VSTOCKS`、走勢模型 `advanceMarketDay()`（大盤+動量 regime+個股波動）、勝率控管（新聞 `computeNews` + 新手/連敗暗助）、`S.market`。設計文件見 `股市小兵聖戰_企劃與數學模型.md`。
- **聊天**：`openChat()` 串接後端 `/api/chat`（Gemini）。首頁右上 💬 FAB（只在 home 場景顯示，`updateChatFab`）。
- **場景 carousel**：`SceneCarousel`（餐廳↔首頁↔寢室），手勢門檻 20px 防誤觸，`_measure()` 每次觸控重量寬度（修登入後 0 寬 bug）。

## 已知注意事項 / 踩過的雷
1. **多人同時改同一份未 commit 的檔**：我（Claude 對話）和 Cursor 都在改 `index.html`/`assets.js`，常常混在一起。**動工前先看目前檔案實況**，別假設是自己上次留下的樣子。
2. **TDZ 白畫面**：`const`（如 VSTOCKS）在宣告前被呼叫會整段 script 中斷→全白。初始化要延遲到宣告之後。
3. **sprite 切圖**：sheet 若非正方形格子（例：512×512 但 4×3），用 `background-size:400% 300%` 塞正方框會變形。要讓格子=正方（例 512×384 的 4×3）。
4. **登入/Render**：Google 登入要在 Render 後台設 `GOOGLE_CLIENT_ID`/`JWT_SECRET`，且 Google Console 要把 Render 網址加進「已授權 JavaScript 來源」。沒設 `GOOGLE_CLIENT_ID` 時 `/api/config` 回 `authEnabled:false`，前端會直接跳過登入 gate。
5. **金鑰**：`.env` 永不 commit（已 gitignore）；密鑰在 Render 環境變數設定。
